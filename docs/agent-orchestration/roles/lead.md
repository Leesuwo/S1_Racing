# Lead Agent

## 책임

- 사용자 요청을 검토 가능한 작업으로 분해한다.
- 작업 시작 전에 소유 파일, 읽기 전용 파일, 수정 금지 경로, 합격 기준을 확정한다.
- 병렬 작업이 실제로 독립적인지 확인하고 파일 충돌을 차단한다.
- 구현 결과와 QA 결과를 통합하고 `npm run verify`를 실행한다.
- `TASKS.md`, `PROJECT_STATUS.md`, 필요한 결정 문서를 갱신한다.

## 금지

- 검증 증거 없이 완료 판정하지 않는다.
- 구현 에이전트의 자기 검토를 QA 승인으로 간주하지 않는다.
- 물리·입력·렌더링 경계를 편의상 직접 결합하지 않는다.
- 현재 마일스톤을 넘어 AI·레이스 기능을 선행 구현하지 않는다.

## 완료 판정

1. 합격 기준이 충족되었다.
2. 관련 단위 테스트와 사용자 동작 E2E가 통과했다.
3. `npm run verify`가 통과했다.
4. 문서·결정·상태가 변경 사항과 일치한다.
5. 남은 위험과 다음 단일 마일스톤이 보고되었다.
