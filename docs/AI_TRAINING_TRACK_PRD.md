# PRD — AI Training Circuit & Deterministic Evaluator

## 문서 정보

- 버전: 0.1.3
- 상태: 구현 기준
- 작성일: 2026-07-24
- 선행 기준: Milestone 1F — Input presets and test track
- 대상 마일스톤: M2A-0 — AI Training Circuit & Evaluator

## 1. 결론

단일 AI 상대를 레이스 세션에 연결하기 전에, AI가 반복 가능한 트랙에서 주행 데이터를 만들고 튜닝값을 검증하는 교육 단계를 둔다. 이 단계의 교육은 초기에는 머신러닝 모델 학습이 아니라 **120Hz 결정적 시뮬레이션, 파라미터 탐색, 회귀 평가**를 뜻한다.

교육 트랙은 Silverstone의 고속·기술적 주행 특성만 참고한 독창 트랙으로 만든다. 실제 Silverstone의 레이아웃, 코너명, 좌표, 시설, 브랜딩, 외부 모델을 복제하지 않는다.

## 2. 문제

현재 `SingleOpponentAI`는 레이싱 라인, 목표 속도, 제동점에 따라 `VehicleControlInput`을 생성할 수 있지만 다음을 체계적으로 측정할 수 없다.

- 코너별 목표 속도와 실제 속도의 오차
- 제동 시작점과 코너 진입 속도
- 레이싱 라인 횡오차와 트랙 이탈
- 튜닝값 변경 전후의 주행 품질
- 동일 입력·동일 상태의 재현성

이 검증 없이 다차량 레이스를 추가하면 AI의 주행 불안정성과 세션 운영 문제가 섞여 원인을 분리하기 어렵다.

## 3. 목표

1. AI가 플레이어와 동일한 `VehicleControlInput` 및 `VehicleSimulation` 경로를 통과하는지 검증한다.
2. 교육 트랙의 중심선, 곡률, 목표 속도, 제동점, 체크포인트를 단일 데이터 원본으로 제공한다.
3. 고속 직선, 강한 제동, 고속 복합 코너, 중속 코너, 저속 탈출을 포함한 커리큘럼을 만든다.
4. 결정적 에피소드 실행과 수치화된 평가 보고를 제공한다.
5. 튜닝값을 범용 AI 코드가 아니라 타입이 있는 설정과 트랙 데이터에 둔다.

## 4. 제외 범위

- 신경망·강화학습 모델 및 학습 프레임워크 도입
- 실버스톤 실제 레이아웃·좌표·코너명·시설물·브랜딩 복제
- 충돌 회피, 추월, 방어, 다차량 순위
- 퀄리파잉·레이스 운영
- 실제 차량 또는 실제 트랙 재현을 의미하는 성능 주장
- 트랙 데이터에서 GLB를 런타임 추정하는 방식

## 5. 교육 트랙 제품 정의

### 5.1 작업명

- 가칭: `Northfield GP`
- 제품 화면과 코드에는 Silverstone 명칭을 사용하지 않는다.
- 최종 명칭은 트랙 데이터와 UI가 안정화된 뒤 결정한다.

### 5.2 주행 특성

Silverstone 공식 자료가 설명하는 고속 직선과 기술적 코너의 대비를 추상적 설계 입력으로만 사용한다. 공식 자료는 GP·National·International 등 복수 레이아웃을 제공하므로, 특정 레이아웃을 재현하지 않고 다음 특성의 조합을 새로 구성한다.

- 긴 가속 구간 뒤의 강한 제동
- 시야 확보가 어려운 고속 복합 코너
- 연속 좌우 코너에서의 안정적인 레이싱 라인 추적
- 중속 코너의 목표 속도 유지
- 저속 코너 탈출 시 트랙션과 변속 제어

### 5.3 독창성 경계

- 실제 트랙의 중심선, 곡선, 폭, 코너 순서, 길이 비율을 좌표 데이터로 전사하지 않는다.
- 공식 코너명·패독·관중석·타워·브리지·표지판·리버리·로고를 사용하지 않는다.
- 외부 모델이나 지도 이미지를 게임 자산으로 가져오지 않는다.
- 참고 링크와 참고한 추상 특성은 이 문서와 라이선스 검토 기록에 남긴다.

## 6. 데이터 계약

현재 프로토타입의 `TestTrackDefinition` 경계를 확장하되, 물리 계층이 트랙별 예외를 알지 않게 한다.

필수 데이터:

- 중심선 샘플과 구간 ID
- 트랙 폭, 표면, 곡률, 선택적 고도·뱅킹
- 섹터와 순서가 있는 체크포인트
- 시작 포즈와 리셋 포즈
- AI 레이싱 라인 점의 위치·방향·목표 속도
- 선택적 제동 진입점과 제동 강도

현재 구현은 `src/tracks/NorthfieldGP.ts`의 `NORTHFIELD_GP_DATA`를 사용한다. 중심선과 `trackWidthM`은 물리 노면 샘플링·디버그 도로 표시가 공유하고, 별도 `racingLine`은 AI 조향·속도·제동 목표를 제공한다.

초기 평면 수치와 AI 튜닝값은 `initial_assumption`이며, 주행감은 `simulation_required`로 표시한다.

## 7. 교육 실행

### 7.1 커리큘럼

1. 직선 가속: 스로틀, 변속, 유한 상태
2. 단일 강제동 코너: 제동 진입점과 코너 속도
3. 고속 복합 코너: Pure Pursuit 조향과 횡오차
4. 중속·저속 코너: 탈출 가속과 트랙션
5. 전체 랩: 섹터 통과, 리셋, 결정성 리플레이

### 7.2 실행 방식

- 모든 에피소드는 120Hz 고정 스텝으로 실행한다.
- 동일 트랙·차량 설정·AI 설정·시드에서 동일한 텔레메트리 해시를 생성해야 한다.
- 초기 튜닝은 설정값 grid search 또는 제한된 반복 탐색으로 구현한다.
- 사용자가 별도 탐색 버튼을 누르지 않는다. 새 교육은 현재 설정으로 에피소드를 끝까지 실행한 뒤, 사용자가 완료한 시나리오 범위에서 후보를 결정적으로 비교한다. 더 낮은 점수의 후보만 다음 에피소드에 자동 적용하며, 일시정지 재개에는 재탐색하지 않는다.
- 고속 코너에서 AI는 차체 슬립이 0.06 rad(약 3.4°) 미만일 때만 요청 스로틀을 유지한다. 한계를 넘으면 lift와 속도 벡터 복구 조향을 사용하며, 자동 후보 평가는 상한 초과 후보를 적용하지 않는다. 이 값은 실제 F1 차량 재현이 아닌 `initial_assumption`이다.
- 차량이 도로 리밋을 벗어난 경우에는 이탈 지표를 계속 수집한다. 차량이 `outerBounds` 맵 경계를 완전히 벗어난 경우에는 즉시 실패로 종료하고, 위치·경계 초과 거리·속도·횡오차·입력을 실패 사례로 보존해 자동 후보 평가의 미완주 패널티에 반영한다.
- 선택된 설정은 버전이 있는 JSON 또는 타입 설정으로 저장하며, 런타임에 임의로 변경하지 않는다.
- 학습·평가 실행기는 React, R3F, DOM에 의존하지 않는다.

## 8. 평가 지표

| 지표 | 목적 | 실패 신호 |
|---|---|---|
| 랩 시간 | 전체 주행 효율 | 비정상적으로 느리거나 에피소드 중단 |
| 트랙 이탈 횟수 | 경계 준수 | 코너에서 반복적인 이탈 |
| 레이싱 라인 횡오차 RMS/P95 | 조향 안정성 | 고속 구간 진동·누적 편차 |
| 목표 속도 오차 RMS/P95 | 속도 제어 | 직선 과속 또는 코너 저속 |
| 제동 초과량 | 제동 시점 | 브레이크 마커 이후에도 과속 |
| 입력 채터링 | 제어 품질 | 스로틀·브레이크·조향의 반복 전환 |
| 차체 슬립 최대값 | 드리프트 방지 | 0.06 rad(약 3.4°) 초과 |
| 결정성 해시 | 재현성 | 동일 조건에서 결과 불일치 |

트랙별 통과 기준은 교육 트랙 데이터와 함께 선언한다. 범용 AI 컨트롤러에 트랙별 임계값을 하드코딩하지 않는다.

## 9. 합격 기준

- AI가 위치·속도·그립을 직접 변경하지 않고 입력만 출력한다.
- 같은 상태와 `dt`에서 동일한 입력을 출력한다.
- 교육 트랙 전체 랩에서 모든 텔레메트리가 유한하다.
- 고속·강제동·복합 코너·저속 탈출 각 시나리오가 독립적으로 평가된다.
- 제동점, 목표 속도, 레이싱 라인 체크포인트가 반복 실행마다 동일하다.
- 전체 랩의 퍼센트는 실제 차량 위치를 레이싱 라인에 투영한 주행 거리로 계산하며, 역주행하면 감소한다. 100%와 완료 상태는 결승선 재통과에서만 발생한다.
- Training Lab 맵은 레이싱 라인을 연속 cyan 곡선으로 표시하고, 데이터 정의 에이펙스는 violet `APEX`, 제동 시작점은 amber `BRAKE`, 현재 AI 목표점은 cyan 링으로 구분한다. HUD의 횡오차 RMS/P95와 함께 라인 추종을 관찰한다.
- 고속 코너에서는 HUD의 차체 슬립과 3.4° 한계를 함께 표시한다. 안정적으로 라인을 따라갈 때만 풀스로틀을 유지하며, 슬립 초과 시 lift가 우선한다.
- 튜닝 결과와 평가 보고를 다시 실행할 수 있다.
- 단위 테스트, 아키텍처 검사, 빌드, 관련 E2E를 포함한 `npm run verify`가 통과한다.

## 10. 위험과 대응

| 위험 | 대응 |
|---|---|
| 실제 트랙 복제로 식별되는 결과물 | 독창 레이아웃·명칭·시설·자산을 사용하고 원본과 좌표 비교를 하지 않는다. |
| AI가 특정 트랙에만 과적합 | 여러 곡률·속도·노면 구간과 별도 검증 시나리오를 사용한다. |
| 튜닝값이 물리 오류를 가림 | AI 보너스 금지, 동일 `VehicleSimulation`, 물리 텔레메트리 독립 검증을 유지한다. |
| 탐색 결과가 비결정적 | 고정 시드, 고정 스텝, 설정 스냅샷, 결과 해시를 저장한다. |
| 교육 실행기가 브라우저에 결합 | 순수 TypeScript 실행기와 브라우저 HUD를 분리한다. |
