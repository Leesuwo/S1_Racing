# QA 보고서 — AI Training Circuit 현재 진행 상태

## 문서 정보

- 작성일: 2026-07-24
- 대상: Milestone 2A-0 — AI Training Circuit & Evaluator
- 범위: Northfield GP 교육 트랙, AI 주행 제어, 자동 튜닝, Training Lab HUD, 자동화 검증
- 판정: 로컬 기능 검증 통과, 인수·동기화 미완료

## 결론

현재 구현은 고속 코너의 과도한 차체 슬립 억제, 실제 레이싱 라인 거리 기반 랩 진행률,
자동 튜닝 흐름, 레이싱 라인·에이펙스·제동점 시각화를 포함한다. 타입 검사, 단위 테스트,
아키텍처 검사, 프로덕션 빌드 및 Playwright E2E는 통과했다.

다만 트랙 리미트·코너 커팅을 무효 랩으로 처리하는 규칙, 특정 목표 랩타임의 인수 규칙,
튜닝 결과의 버전 저장이 구현되지 않았다. 이 세 항목이 완료되기 전에는 M2A-0를 완료나
배포 가능 상태로 판정하지 않는다.

## 발견 사항

### P1 — 트랙 리미트 및 코너 커팅 무효화가 없다

`AITrainingRunner`는 도로 밖 위치를 `offTrackCount`로 누적하지만, 맵의
`outerBounds` 바깥으로 완전히 나가지 않으면 에피소드를 실패시키지 않는다. 따라서
짧은 트랙 이탈이나 코너 커팅은 감점일 뿐 유효 랩을 무효로 만들지 않는다.

**영향:** “트랙 리미트와 코너 커팅 없이 목표 랩타임 달성”이라는 교육 목표를 현재
점수만으로 보장할 수 없다.

**조치:** 매 fixed step에서 도로 경계를 넘으면 `invalidLap`과 무효 사유를 기록한다.
무효 랩은 결승선을 통과해도 성공으로 처리하지 않고, 자동 튜닝 후보 선택에서도 제외한다.

### P1 — 목표 랩타임 인수 조건이 없다

전체 랩 진행률은 레이싱 라인 투영 거리로 정확히 계산되지만, 목표 랩타임, 실제 랩타임,
델타 및 성공·실패 기준은 아직 없다.

**영향:** AI가 단순히 완주했는지와 사용자가 요구한 성능 목표를 충족했는지를 구분할 수 없다.

**조치:** 시나리오 데이터에 목표 랩타임을 선언하고, 결승선 재통과 시 유효 랩의 실제 시간과
델타를 평가한다. 무효 랩 또는 목표 시간 초과 후보는 자동 적용 대상에서 제외한다.

### P1 — 튜닝 결과가 재현 가능한 파일로 저장되지 않는다

화면의 자동 튜닝 결과와 적용된 AI 설정은 런타임 메모리에만 존재한다. 시드, 설정 스냅샷,
평가 텔레메트리, 결정성 해시를 보존하는 버전 결과 파일이 없다.

**영향:** 새로고침 또는 새 세션 뒤에 어떤 학습 결과가 더 나았는지 감사하거나 재현할 수 없다.

**조치:** 버전·시드·트랙 데이터 버전·AI 설정·평가 결과·결정성 해시를 포함한 결과 JSON을
저장하고, 결과를 다시 읽어 평가를 재실행하는 검증을 추가한다.

### P2 — E2E 시나리오 보강이 필요하다

현재 E2E는 전체 랩 진행률, 저속 탈출, 고속 시나리오의 슬립 HUD, 자동 튜닝 흐름 등을
검증한다. 하지만 브레이킹 시나리오 선택, 코너 커팅 무효화, 목표 랩타임 성공·실패는
브라우저 흐름으로 검증하지 않는다.

**조치:** 유효 랩 기능과 함께 다음 시나리오를 E2E에 추가한다.

1. 제동 시나리오의 시작·완료·자동 튜닝 결과
2. 트랙 이탈 뒤 무효 랩 사유와 자동 적용 제외
3. 목표 랩타임 이내·초과의 성공/실패 및 델타 HUD

### P2 — 프로덕션 번들 크기 경고가 남아 있다

빌드 산출 JavaScript는 약 3.5 MB, gzip 기준 약 1.2 MB이며 청크 크기 경고가 발생한다.

**조치:** 기능 인수 후 렌더링·물리 의존성의 코드 분할 후보를 프로파일링한다. 현재는 기능
정확성보다 우선하지 않으며, 성능 측정 없이 분할하지 않는다.

### P3 — Rapier 초기화 경고가 반복된다

테스트 중 Rapier 초기화 파라미터의 deprecated 경고가 발생하지만, 테스트 실패는 아니다.

**조치:** 다음 물리 확장 전 현재 라이브러리 API에 맞게 초기화 옵션을 정리한다.

## 다음 단일 마일스톤

**M2A-0 인수 — 유효 랩과 목표 랩타임**

1. 트랙 리미트 이탈을 즉시 무효 랩으로 전환하고 사유를 HUD·결과에 표시한다.
2. 체크포인트 순서와 결승선 재통과를 유효 랩 완료 조건으로 유지한다.
3. 시나리오별 목표 랩타임·실제 시간·델타를 평가 및 자동 튜닝 선택 기준에 추가한다.
4. 제동, 코너 커팅, 목표 시간 성공·실패를 단위 테스트와 E2E로 검증한다.
5. 결과 스냅샷 저장까지 완료한 뒤 전체 게이트를 통과시키고 커밋·푸시한다.

## 남은 위험

- 현재 도로 경계 판정은 평면 차량 위치 샘플에 기반한다. 향후 차량 접지 폭·연석·충돌을
  도입하면 트랙 리미트 규칙을 휠별 접지점 기준으로 확장해야 한다.
- 차체 슬립 3.4°는 실차 재현값이 아닌 초기 가정이다. 실제 차량 데이터로 해석하지 않으며,
  반복 시뮬레이션과 사용자 플레이테스트로만 조정한다.
