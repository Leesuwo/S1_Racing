# Milestone 1F — Input presets and test track

상태: 완료 (2026-07-22)

Milestone 1E의 결정적 차량 물리 위에 여러 입력 장치와 반복 가능한 테스트 트랙 데이터를 연결한다. 입력 장치는 모두 `VehicleControlInput`으로 정규화하고, 물리와 월드 시각화는 동일한 트랙 데이터를 읽는다.

## 목표

- 키보드·마우스·게임패드·휠 입력을 동일한 물리 입력 경계로 제공한다.
- 트랙 구간·노면·브레이크 마커·체크포인트·시작 포즈를 데이터로 고정한다.
- 입력 지연, 리셋, 트랙 경계 상태를 자동 검증한다.

## 구현 내용

### 입력 프리셋

- `InputPreset.ts`에 네 가지 프리셋과 휠 축 캘리브레이션 계약을 추가했다.
- `BrowserVehicleInput`은 브라우저 API를 계속 독점하고, 선택된 장치를 공통 `VehicleControlInput`으로 변환한다.
- 게임패드는 표준 축·트리거·범퍼를 사용하며, 변속은 버튼 상승 에지에서 한 번만 예약한다.
- 휠은 장치 ID·비표준 매핑을 best-effort로 찾고, 조향·가속·브레이크 축을 캘리브레이션 범위로 정규화한다.
- 게임패드·휠이 연결되지 않은 경우 키보드 페달 입력을 안전한 폴백으로 사용한다.
- UI에서 프리셋을 전환할 수 있고, 버튼과 `R` 키로 시작 포즈 리셋을 요청한다.

### 반복 가능한 테스트 트랙

`src/tracks/TestTrack.ts`가 다음 데이터를 단일 원본으로 소유한다.

- 외곽 경계와 인필드 잔디 경계
- 스타트 직선·우측 코너·백 스트레이트·좌측 코너 구간
- 스타트/피니시와 100m·50m 브레이크 마커
- 순서가 고정된 네 개 체크포인트
- 시작 위치 `(-10, 10)`과 yaw `π/2`

물리 표면 샘플러와 R3F 월드 시각화는 이 데이터에서 각각 노면과 마커를 읽는다. 인필드 잔디와 외곽 경계 바깥은 주행 표면 이탈로 판정하고, 외곽 경계 바깥은 `off-track` 섹션으로 구분한다.

## 합격 기준

- 네 입력 프리셋이 공통 `VehicleControlInput` 범위로 변환된다.
- 게임패드 변속 에지가 중복 입력되지 않는다.
- 휠 조향·페달 축이 캘리브레이션과 `[0, 1]` 페달 계약을 따른다.
- 시작 포즈·구간·마커·체크포인트 순서가 반복 실행마다 동일하다.
- 첫 입력 샘플에서 가속 입력이 반영된다.
- 리셋 후 속도·yaw·위치가 데이터 정의 시작 포즈로 돌아온다.
- 트랙 외곽 이탈 시 HUD가 `이탈 · 리셋 권장`을 표시한다.
- 단위 테스트, 아키텍처 검증, 프로덕션 빌드, 브라우저 E2E가 통과한다.

## 제외 범위

- AI 레이서와 레이싱 라인 제어
- 다차량 세션·퀄리파잉·레이스 운영
- 실제 트랙 복제, 높이·연석·벽 충돌
- 휠 장치별 제조사 SDK와 자동 영구 캘리브레이션 저장
