# Decisions

## 2026-07-16 — D-007

**결정:** Milestone 1B에서는 휠 운동학을 순수 TypeScript 모듈로 분리하고, 각 휠의 장착점·접지점·조향 방향·접지점 속도를 Rapier 차체 상태에서 계산한다.

**이유:** 타이어 힘 모델이 Rapier에 적용되기 전에 좌표계·조향 부호·yaw-rate 항을 독립적으로 검증해야 한다. 이 모듈은 React와 Rapier API에 의존하지 않아 AI와 자동 물리 검증에서도 재사용할 수 있다.

**포함:** 전륜 조향, 4개 휠 장착점, 차체 선속도와 각속도에 따른 접지점 속도, 차체 yaw·속도의 Rapier 동기화, 조향 HUD.

**제외:** 휠 회전 관성, 종·횡방향 타이어 힘, 슬립 비율·슬립 각 기반 그립, 실제 연석 충돌.

## 2026-07-16 — D-006

**결정:** Milestone 1A에서는 Rapier 동적 차체와 네 개의 하향 레이캐스트 서스펜션을 도입하되, X/Z 구동력과 yaw는 기존 순수 TypeScript 평면 차량 물리가 계속 소유한다.

**이유:** 현재 검증된 조작감·입력 경계를 보존하면서 실제 Raycast 접지, 스프링·댐퍼, 차체 높이를 독립적으로 검증하기 위해서다. Rapier와 평면 적분기가 동시에 종·횡방향 운동을 계산하면 두 물리 계층이 충돌한다.

**포함:** Rapier 정적 지면, 동적 차체, 4개 접지 레이, 120Hz 외력 초기화, 정적 차체 안정성 테스트, 접지·차체 높이 HUD.

**제외:** 종·횡방향 타이어 힘의 Rapier 적용, 실제 트랙 collider, 연석·경사, 충돌 손상, 롤·피치 기반 차체 거동.

## 2026-07-16 — D-005

**결정:** Rapier 연결 전 순수 TypeScript 평면 차량 물리로 `Physics Prototype v0.1`을 구현한다.

**이유:** 타이어·입력·고정 스텝의 주행 감각을 브라우저에서 빠르게 검증하고, 물리 계층을 React/R3F에서 독립시키기 위해서다.

**포함:** 엔진 구동력, 브레이크, 슬립각 기반 횡력, 결합 그립, 다운포스, 항력, 아스팔트·잔디 표면.

**제외:** 실제 타이어 열·마모, AI, 퀄리파잉, 레이스. Rapier 4휠 접지는 D-006에서 별도 도입한다.

## 2026-07-16 — D-001

**결정:** Milestone 0은 차량 물리 구현이 아니라 실행·검증 기반으로 제한한다.

**이유:** 수치가 검증되지 않은 상태에서 차량·AI·트랙을 동시에 만들면 잘못된 물리값과 구조가 누적된다.

## 2026-07-16 — D-002

**결정:** WebGL2를 초기 렌더링 기준으로 사용하고 WebGPU 전환은 후속 조사로 둔다.

**이유:** 브라우저 호환성을 우선하고 실제 병목을 측정한 뒤 렌더러 선택을 재평가한다.

## 2026-07-16 — D-003

**결정:** React UI와 물리 코드를 분리하고 `VehicleControlInput`을 입력 경계로 사용한다.

**이유:** 플레이어·AI·게임패드·휠 입력을 동일한 물리 API로 연결하고 결정성을 유지하기 위해서다.

## 2026-07-16 — D-004

**결정:** 프로젝트 전용 검증 스킬은 `.agents` 권한이 확보된 뒤 추가한다.

**이유:** 현재 작업 폴더의 `.agents`는 읽기 전용이며, 자동 검색되지 않는 임의의 위치에 스킬을 만들지 않는다.
