# Decisions

## 2026-07-19 — D-011

**결정:** 공력은 순수 `AeroModel`에서 속도 제곱 기반 전후 다운포스·드래그를 계산하고, Rapier 고정 스텝이 전·후 차축과 차체 중심에 힘으로 적용한다.

**이유:** 공력 계수와 노면 드래그 배율을 React나 Rapier API와 분리해야 단위 테스트와 자동 검증에서 같은 계산을 재사용할 수 있다. 전후 적용점은 서스펜션 하중과 차체 자세 응답을 실제 강체에 전달한다.

**검증:** 정지 시 공력 0, 속도 2배 시 공력 4배, Rapier 텔레메트리 양수, 결정적 직선·코스트다운·유한 상태 검증을 사용한다.

## 2026-07-19 — D-010

**결정:** 구동계는 순수 `Drivetrain` 명령으로 분리하고, Rapier에는 후륜 구동 토크·엔진 브레이크 토크·휠 각속도 피드백만 전달한다.

**이유:** 기어비·토크 커브·RPM·클러치·엔진 브레이크를 고정 120Hz에서 결정적으로 계산하면서, 타이어 힘과 휠 회전 적분은 Rapier 접지 경계에서 한 번만 수행해야 한다.

**검증:** 1단/8단 토크·RPM 차이, 연결 주행 중 엔진 브레이크, 정지 상태 엔진 브레이크 0, 후륜 휠 각속도와 차량 속도 통합 테스트를 사용한다.

## 2026-07-19 — D-009

**결정:** Rapier 차체 collider는 설정된 차량 질량을 collider 체적의 양의 밀도로 반영하고, 0 밀도 collider로 인해 회전 관성이 0이 되는 구성을 사용하지 않는다.

**이유:** 기존 구성은 차체 질량은 780kg으로 표시되지만 주 관성 모멘트가 `(0, 0, 0)`으로 계산되어 접지점 횡력이 있어도 yaw가 발생하지 않았다. collider 밀도로 질량과 형상 기반 관성을 함께 생성하면 현재 차체 질량을 유지하면서 회전 응답을 검증할 수 있다.

**검증:** 정지 상태 조향은 회전하지 않고, 주행 중 우회전 조향과 후륜 구동을 적용하면 Rapier quaternion과 Y축 angular velocity가 우회전 방향으로 변하는 통합 테스트를 사용한다.

## 2026-07-16 — D-008

**결정:** Milestone 1C부터 휠별 슬립 비율·슬립 각·하중 민감도·결합 그립을 순수 `TireModel`로 계산하고, Rapier 차체의 실제 Raycast 접지점에 종·횡방향 타이어 힘을 적용한다.

**이유:** 타이어 한계와 가속·제동·조향의 반응이 동일한 차체 적분기에서 계산되어야 접지점 힘과 yaw가 일관되게 검증된다. `VehicleSimulation`은 기존 입력·기어·RPM 경계를 보존하되, 매 고정 스텝 뒤 Rapier 결과로 평면 스냅샷을 동기화한다.

**포함:** 휠 회전 속도, 후륜 구동 토크의 초기 변환, 앞 58%·뒤 42% 제동 배분, 저속 분모 보호, 하중 민감도, 결합 그립, 슬립·그립 HUD와 Rapier 가속·제동 통합 테스트.

**제외:** 실차 Magic Formula 계수, 실제 타이어 열·마모·압력, 디퍼렌셜, 엔진 브레이크와 완전한 구동계 토크 모델, 경사·연석·충돌.

## 2026-07-16 — D-007

**결정:** Milestone 1B에서는 휠 운동학을 순수 TypeScript 모듈로 분리하고, 각 휠의 장착점·접지점·조향 방향·접지점 속도를 Rapier 차체 상태에서 계산한다.

**이유:** 타이어 힘 모델이 Rapier에 적용되기 전에 좌표계·조향 부호·yaw-rate 항을 독립적으로 검증해야 한다. 이 모듈은 React와 Rapier API에 의존하지 않아 AI와 자동 물리 검증에서도 재사용할 수 있다.

**포함:** 전륜 조향, 4개 휠 장착점, 차체 선속도와 각속도에 따른 접지점 속도, 차체 yaw·속도의 Rapier 동기화, 조향 HUD.

**제외:** 휠 회전 관성, 종·횡방향 타이어 힘, 슬립 비율·슬립 각 기반 그립, 실제 연석 충돌.

## 2026-07-16 — D-006

**결정:** Milestone 1A에서는 Rapier 동적 차체와 네 개의 하향 레이캐스트 서스펜션을 도입하되, X/Z 구동력과 yaw는 기존 순수 TypeScript 평면 차량 물리가 계속 소유한다.

**이유:** 현재 검증된 조작감·입력 경계를 보존하면서 실제 Raycast 접지, 스프링·댐퍼, 차체 높이를 독립적으로 검증하기 위해서다. Rapier와 평면 적분기가 동시에 종·횡방향 운동을 계산하면 두 물리 계층이 충돌한다.

**포함:** Rapier 정적 지면, 동적 차체, 4개 접지 레이, 120Hz 외력 초기화, 정적 차체 안정성 테스트, 접지·차체 높이 HUD.

**제외:** 종·횡방향 타이어 힘의 Rapier 적용, 실제 트랙 collider, 연석·경사, 충돌 손상, 롤·피치 기반 차체 거동.
